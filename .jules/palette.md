
## 2024-05-24 - Visual Required Indicators vs Native HTML5 Validation
**Learning:** Adding the native HTML5 `required` attribute to inputs is excellent for screen reader accessibility, but browsers will automatically display their own tooltip validation UI (e.g., "Please fill out this field") when a user tries to submit an empty form. If the application already has a custom React-based validation state and error message display (e.g., rendering a red error box below the form), the native tooltips can clash or prevent the custom UI from showing.
**Action:** When retrofitting accessibility using the native `required` attribute on a form that already handles its own visual error states, always add the `noValidate` attribute to the `<form>` element. This instructs the browser to skip its visual validation UI, allowing the custom React logic to handle the presentation, while still retaining the semantic accessibility benefits of the `required` inputs for assistive technologies. Also, ensure asterisks used as visual indicators have `aria-hidden="true"` so they aren't redundantly announced by screen readers.
