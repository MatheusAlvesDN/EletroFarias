## 2024-05-24 - Login Form Error Accessibility
**Learning:** In `integra-front`, the login form is duplicated across `src/app/page.tsx` and `src/app/homepage/page.tsx`. This means any UX or accessibility improvements made to one form must be manually mirrored to the other to ensure consistency across entry points. Furthermore, ensuring asynchronous validation errors are announced by screen readers requires pairing `role="alert"` and `aria-live="assertive"` on the error container with `aria-invalid` and `aria-describedby` on the corresponding input fields.
**Action:** When updating any UI component in `integra-front`, always check for duplicated code (like the login form) and ensure changes are applied to all instances. For form validation, consistently use ARIA attributes to link errors to inputs.
