## 2024-05-15 - Form Accessibility and Interaction Polish
**Learning:** Adding native `required` validation and `disabled` states to form inputs improves accessibility and prevents duplicate submissions during loading, but when combined with a custom UI error state, standard browser tooltips override the UI. NextJS applications handle alerts nicely when given specific `aria-live` and `role` attributes on the error container itself.
**Action:** Always add `noValidate` to the `<form>` to suppress default browser popups when custom errors are displayed. Include `disabled={loading}` on all form inputs and clearly mark disabled UI states with Tailwind `disabled:opacity-60 disabled:cursor-not-allowed` so users understand the loading state. Add `aria-live="assertive"` and `role="alert"` to any custom error container to ensure immediate screen reader announcement.
