## 2024-05-19 - Accessible Form Validation Error Pattern
**Learning:** Adding `role="alert"`, `aria-live="assertive"`, and `id` to error message containers, while linking inputs using `aria-invalid` and `aria-describedby`, ensures screen readers announce validation failures effectively in Next.js/React login forms.
**Action:** When working on form validations across `integra-front`, consistently apply this pattern for synchronous and asynchronous error messaging to ensure compliance and robust accessibility.
