## 2024-05-24 - Hardcoded JWT Fallback Secret
**Vulnerability:** The `JWT_SECRET` in both `jwt.strategy.ts` and `auth.module.ts` relied on insecure fallbacks (`process.env.JWT_SECRET || 'dev-secret'`) and optional variables without explicit failure states.
**Learning:** Hardcoded default secrets in authentication modules allow attackers to easily forge valid JWTs if the default string is known and environment variables fail to load in production. Additionally, automated linting tools (`eslint --fix`) triggered by generic test commands can unintentionally rewrite the entire codebase, masking surgical security fixes with thousands of formatting changes.
**Prevention:** Always enforce fail-secure defaults by throwing an explicit exception (`throw new Error('JWT_SECRET must be defined')`) during application or module initialization if critical secrets are missing. Never rely on insecure string fallbacks. When committing security fixes, avoid generic `git add .` to prevent unrelated automated formatting changes from polluting the PR diff; explicitly stage only the modified files (e.g., `git add src/auth/jwt.strategy.ts`).
