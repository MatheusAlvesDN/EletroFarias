## 2025-05-03 - Hardcoded JWT Secret Fallback Removed
**Vulnerability:** The application was using `process.env.JWT_SECRET || 'dev-secret'` inside `jwt.strategy.ts` and relying on the module scope evaluation of `process.env.JWT_SECRET` in `auth.module.ts`. This meant that if `JWT_SECRET` wasn't correctly loaded during module initialization (a common race condition with `ConfigModule`), the app could silently fall back to 'dev-secret' in production, causing a critical vulnerability.
**Learning:** In NestJS applications, environment variables might not be fully available at the top-level scope of module decorators when using `process.env`. Asynchronous module configuration (`registerAsync`) should be used with injected configuration services.
**Prevention:** Always use `JwtModule.registerAsync` and inject `ConfigService` to safely retrieve and validate environment variables. Throw an explicit error if critical secrets are missing rather than providing a default fallback. Use `ConfigService` inside the constructor of strategies instead of module-level `process.env`.
