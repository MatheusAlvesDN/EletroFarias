## 2025-03-12 - Prevent insecure secret fallback in JWT configuration
**Vulnerability:** The application was passing `process.env.JWT_SECRET` directly to `JwtModule.register` and falling back to a hardcoded string `'dev-secret'` in `JwtStrategy`. This could lead to a critical security issue where the application uses an insecure default secret in production if `JWT_SECRET` is not set or misconfigured.
**Learning:** Using synchronous `process.env` lookups or default fallbacks allows the application to start in an insecure state without warning. To ensure strict validation, explicit dependency injection using `ConfigService` with asynchronous setup allows for centralized configuration validation before dependent modules (like `JwtModule`) are configured.
**Prevention:** In NestJS applications, always use `ConfigService` (and its module integration like `JwtModule.registerAsync`) to inject environment variables, and explicitly throw errors during configuration if critical secrets like `JWT_SECRET` are missing, instead of using a weak or default fallback. Avoid global code reformatting when applying surgical security fixes.
