## 2024-04-04 - Secure JWT Strategy Initialization
**Vulnerability:** The application was falling back to a hardcoded JWT secret ('dev-secret') in `JwtStrategy` when `JWT_SECRET` was missing from the environment. This is a critical security risk because it uses a known, weak secret for token signing and validation.
**Learning:** When using NestJS, configuration parameters should be explicitly injected using `ConfigService`. Bypassing dependency injection and directly using `process.env` or providing a hardcoded fallback secret hides configuration failures silently and introduces a severe security flaw. The application should "Fail Securely" rather than quietly relying on an insecure default.
**Prevention:** Configure `JwtModule` with `JwtModule.registerAsync()` to explicitly require `ConfigModule` and `ConfigService`. In `JwtStrategy`, inject `ConfigService`, retrieve the secret, and throw an explicit error if it is not found to prevent successful initialization with a fallback secret.
