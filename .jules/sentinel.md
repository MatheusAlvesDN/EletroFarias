## 2023-11-20 - JWT Secret Hardcoded in Strategy
**Vulnerability:** The JWT strategy (`integra-ifood/src/auth/jwt.strategy.ts`) has a hardcoded default secret (`'dev-secret'`). While the module registers it via `process.env.JWT_SECRET`, the strategy allows fallback to a weak, known secret. If `JWT_SECRET` is missing in production, attackers could forge JWTs and gain full unauthorized access. Furthermore, `app.module.ts` initializes `AuthModule` without `ConfigModule` asynchronous loading, risking `JWT_SECRET` being read before the environment is fully loaded.
**Learning:** `process.env` might be undefined at module initialization in NestJS if `ConfigModule` is not properly used. Falling back to a static dev secret is dangerous in production.
**Prevention:** Do not use hardcoded fallbacks for secrets. Fail fast if secrets are missing. Register modules asynchronously (e.g., `JwtModule.registerAsync`) and use `ConfigService` to ensure environment variables are correctly loaded. Ensure strategies also rely on `ConfigService` or only use `process.env` without fallback.
