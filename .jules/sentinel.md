## 2025-02-13 - [CRITICAL] Fix hardcoded JWT secret fallback and insecure config
**Vulnerability:** The application used a hardcoded fallback (`'dev-secret'`) in `JwtStrategy` and insecurely accessed `process.env.JWT_SECRET` directly in `AuthModule`.
**Learning:** In NestJS, when configuring modules like `JwtModule`, `registerAsync` should be used with `ConfigModule` and `ConfigService` to safely retrieve environment variables. Additionally, strategies like `JwtStrategy` should explicitly throw an error if the secret is missing to "fail securely". If dependencies are injected into the constructor of a class extending `PassportStrategy` (e.g. `constructor(configService: ConfigService)` without access modifiers), it avoids TypeScript TS2376 errors when performing validation before calling `super()`.
**Prevention:** Always use `ConfigService` to retrieve secrets. Never use hardcoded fallback secrets for security-critical operations like JWT signing/verification. Fail fast and securely if critical configuration is missing.
